Font shorthand notation.

Using font shorthand notation we can set the following font properties using just a single declaration:

  • font-style
  • font-variant
  • font-weight
  • font-size
  • line-height
  • font-family

While we don't have to explicitly set each one, we are required to define font-family and font-size each time we use it. If we fail to set a value for any of the remaining font properties, the default value is assigned. This can actually cause other values to be overwritten if care is not taken when writing the syntax.

For example, headings are typically bold by default.

Optional keywords.

You an also use a set of optional keywords which instructs the user agent to use the font formatting used by those system fonts. This is especially useful when creating applications or UI widgets:

  • caption
  • icon
  • menu
  • message-box
  • small-caption
  • status-bar
